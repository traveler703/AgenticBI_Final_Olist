# 演示脚本（5–8 分钟）

## 0. 开场（30s）
- 项目定位：面向业务人员的自然语言 Agentic BI。
- 技术核心：DeepSeek / Ollama 可切换 + LangGraph 多智能体 + MySQL 物理预聚合 + Vite/ECharts 三栏前端。
- 一句话亮点：协调器 ReAct + 数据 Agent 的 Plan-Execute-Reflect-Replan，仓库与运行库物理隔离。

## 1. 环境与数据就绪（40s）
- 展示 Docker MySQL 运行：`docker ps`
- 展示数据路径：`data_layer/data/raw/`（9 个 CSV）
- 强调两个库与两套账号：`olist_bi`（只读 `olist_ro`）/ `agentic_app`（读写 `olist_etl`）

## 2. 预聚合层与性能（1min）
- 展示已建的 11 张 `mv_*` 物理预聚合表
- 运行：`python -m scripts.benchmark_preagg`
- 强调原表 JOIN vs 预聚合的耗时差异（`data/processed/benchmark_report.png`）

## 3. 描述性分析（1min）
- 提问：`2017 年哪个州的销售额最高？`
- 展示右侧面板的命中 SQL（`mv_state_sales`）、命中表与耗时，以及柱状图

## 4. 地理可视化（40s）
- 提问：`在地图上展示各州销售额的地理分布。`
- 展示数据 Agent 命中 `mv_state_geo_sales`（含经纬度）→ 可视化 Agent 选择地理气泡图

## 5. 诊断性分析（1min）
- 提问：`为什么某些州的平均配送时长显著高于全国均值？`
- 展示诊断 Agent 的 haversine 距离下钻散点图与“相关非因果”声明

## 6. 预测性分析（1min）
- 提问：`根据历史趋势预测未来 6 周销售额，并给出趋势解读。`
- 展示历史+预测曲线与 90% 置信区间，强调单周与合计区分、非负约束

## 7. 规范性分析与加分模块（1min）
- 提问：`基于全部分析结果，给出平台 3 个月内的三大优先改进策略。`
- 展示高/中/低优先级、含对象+动作+指标的建议
- 提问：`如果将 Top20 高差评卖家的商品统一下架，平台整体评分预估提升多少？`（What-if）
- 提问：`扫描近期数据，是否有某州订单量骤降或差评率突升的异常？`（异常检测，含异常幅度图）

## 8. 自我纠错与单轮隔离（40s）
- 提问组合型：`2017 年 Top5 各州月度 GMV 趋势`，跟随状态流展示「规划 → 执行 → 反思 → 重规划」，最终恰好 5 条线的分组折线
- 连续提交两个无关问题，展示新问题不继承上一轮筛选条件（仓库只读账号无法访问运行库）

## 9. 收尾（30s）
- 回扣评分点：物理预聚合优化、多 Agent 协作（ReAct/Reflexion/Plan-and-Execute）、四层分析、12 类图表与三栏交互界面、读写物理隔离。
